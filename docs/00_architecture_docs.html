Pipeline Architecture & Orchestration

This document outlines the folder structure of the DM-ESBL clinical pipeline and explains how to use the make.bat manager to execute the workflow.

1. Folder Structure

The project strictly separates configuration, core clinical logic, execution scripts, and generated reports to ensure reproducibility and easy auditing.

dm-esbl/
├── assets/                 # Static clinical knowledge bases
│   └── res195-comorbidity-cci-gold.csv
├── config/                 # YAML files defining how logic is applied
│   ├── feature_config.yaml   # Weights for MEWS, SIRS, Pitt, etc.
│   └── code_search.yaml      # Keyword rules for clinical code discovery
├── data/                   # Patient data (Ignored by Git)
│   ├── raw/                  # Unprocessed synthetic iCARE data
│   └── processed/            # Final engineered dataframes ready for modeling
├── reports/                # Logs and outputs for clinical review
│   ├── score_validation.log
│   └── code_search_results.txt
├── scripts/                # Executable steps in the pipeline (01 to 06)
├── src/                    # Core clinical Python logic (No execution here)
│   ├── phenotypes.py         # Functions to evaluate raw patient conditions
│   └── scores.py             # Functions to calculate complex medical scores
├── tests/                  # Pytest suite protecting the clinical logic
│   ├── cases.csv             # The "Gold Standard" test patients
│   ├── test_phenotypes.py
│   └── test_scores.py
├── make.bat                # Windows Pipeline Manager
└── Makefile                # Linux/macOS Pipeline Manager

2. Using the make.bat Pipeline Manager

Because Windows does not natively support the make utility, we use a custom batch script (make.bat) to orchestrate the pipeline. It automatically activates the correct Python module paths (using python -m) so your scripts can safely access the src/ folder.

Interactive Menu Mode

If you simply run the script without any arguments, it will present an interactive menu:

.\make.bat

You can then type a number (e.g., 1 for all, 5 for validation) to run a specific step.

Direct Command Mode

For faster execution, you can pass the target name directly in your PowerShell terminal. This skips the menu and executes the code immediately.

Command What it does
.\make.bat all Runs steps 1 through 4 sequentially.
.\make.bat generate Runs 01_generate_data_v2.py.
.\make.bat features Runs 02_build_features_icare.py.
.\make.bat evaluate Runs 03_evaluate_scores_v2.py.
.\make.bat validate Runs the clinical audit against cases.csv.
.\make.bat search Generates the ICD/SNOMED keyword report.
.\make.bat test Runs the Pytest suite to check for logic drift.
